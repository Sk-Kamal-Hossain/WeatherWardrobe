Overall Description

2.1 Product Perspective

Weather Wardrobe is a smart and easy solution to wear a smart dress with the weather and to buy the outfits users like. It will serve people with the best outfit to wear and to buy.

2.2 Product Function

The main function of this app is to suggest outfit according to weather forcast and sell them to the users.

  1. Customer Login Panel
  2. Search Product
  3. Live Weather Info
  4. Exclusive Offers
  5. Dress Suggestions
  6. Customer Support

2.3 User Class and Characteristics

The software will consist of mainly two types of users: registered users and the admin. The admin’s job is pretty simple and straightforward, they can only update the store. As the app will suggest dresses, so if the user wants, they can buy those.

2.4 Operating Enviornment

Weather-Wardrobe run on Win7,Win8 , Win10 , Linux .

  • Server: Localhost
  • Operating System: Windows
  • Hard Drive : Minimum 10 GB .
  • RAM : More than 4 GB .
  • Processor : Pentium 4 or higher.
  • Database: MySQL
  • Platform: Python, Django
  • Version Control : Git,Bitbucket

2.5 Design Implementation Constraints

This system is designed in Django framework . Decision regarding which database to use should be taken considering the fact that data being exchanged or stored is large ,and the appropriate data management system will yield efficient performance . Limted to HTTP/HTTPS protocols. We'll use MySQL for database.

5.1 Performance Requirements

The system should be balanced in terms of operating efficiency. It should use a little memory and will be easily accessible by the user. memory management should be done discreetly so that no part of the memory is wasted.

5.2 Safety Requirements

System usage will not pose a risk to human users. It will not violate any political and governmental laws. We respect your privacy and are committed to protecting your information.

5.3 Security Requirements

  • System use secured database.
  • Account ID and password (PIN) protection.
  • Sign-off Button.
  • Failed Log-on Attempts

5.4 Software Quality Attributes

  • We are concerned about system failures.
  • System quality is a way that it can be easily used by the user.
  • The system cares about the quality of conversion that will change.
  • The level of performance relates to response times and similar steps for different events.
  • The quality of this system cares about fast speed, compliance. This is also secure, accurate, and therefore reliable.

5.5 Business Role

  • The user can log in at any time with any number of times and search for various details of his account.
  • All users will have different IDs and passwords to sign in to the account information.
  • The solution offers very simple and modified features.
  • very easy to view and operate various features.
